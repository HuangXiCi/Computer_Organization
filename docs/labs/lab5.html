处理器设计三

处理器的时钟周期完全取决于运行时间最长的那条指令所需要的时间。 因此单周期处理器的数据通路必须能够支持所有指令的功能，这会导致时钟周期较长，处理器性能受限。 我们学习了流水线处理器的原理和设计方法，流水线处理器通过把指令执行过程拆分成多个阶段，并允许多条指令同时处于不同阶段来提高性能。 但是流水线处理器的设计和实现相对复杂，需要处理各种冒险和冲突问题。

多周期处理器则是一种介于单周期处理器和流水线处理器之间的设计方案，它通过把指令执行过程拆分成多个阶段来缩短时钟周期，但在同一时刻仍然只处理一条指令。 由于实验课程的时间限制，我们暂时不考虑流水线处理器的设计和实现，而是从多周期处理器开始练习。 本次实验中，我们将把前面已经实现的单周期处理器改造成多周期处理器。 多周期处理器的基本思想是： 将一条指令拆分成多个阶段执行，每个阶段占用一个时钟周期。 为后续的流水线处理器设计打下基础，感兴趣的同学可以在完成多周期处理器后继续挑战流水线处理器的设计和实现。 欢迎各位同学尝试自行设计流水线处理器，如果有任何问题可以随时交流。

1 从单周期到多周期

单周期处理器的特点是每条指令都在一个周期内完成。 因此，不同类型指令虽然实际需要的功能不同，但它们都必须占用同样长的周期。

例如：

  • add 指令只需要取指、译码、执行和写回；

  • sw 指令需要取指、译码、计算地址和写存储器；

  • lw 指令需要取指、译码、计算地址、读存储器和写回。

在单周期处理器中，所有这些操作都被放在一个周期中完成，导致时钟周期由最慢的指令决定。

多周期处理器是把一条指令拆分成若干阶段，每个阶段使用一个时钟周期完成。

典型的阶段划分如下：

  1. IF：Instruction Fetch，取指；

  2. ID：Instruction Decode，译码并读取寄存器；

  3. EX：Execute，执行运算或计算地址；

  4. MEM：Memory Access，访存；

  5. WB：Write Back，写回。

不同指令所需的阶段数量不同。 例如普通算数运算指令不需要 MEM 阶段，而 sw 指令不需要 WB 阶段。

因此不同类型指令所经历的阶段数不同。例如：

  • 普通算数运算指令通常经历 IF、ID、EX、WB；

  • sw 指令通常经历 IF、ID、EX、MEM；

  • lw 指令通常经历 IF、ID、EX、MEM、WB；

当然你也可以每条指令都经历全部五个阶段，只不过在不需要的阶段不执行任何操作，这样设计会更简单一些。

1.1 多周期处理器中的级间寄存器

在单周期处理器中，一条指令的所有组合逻辑都在一个周期内完成，中间结果不需要保存。 而在多周期处理器中，一条指令会跨越多个时钟周期执行，因此我们必须在阶段之间保存中间结果。 每个周期都需要将之后阶段所需的数据通过寄存器保存起来，以便后续阶段使用。

这些寄存器的作用是把单周期处理器中的长组合逻辑路径切分成多段。 每个阶段只完成其中一部分工作，并把结果保存到寄存器中，供后续阶段使用。

例如 lw rd, imm(rs1) 指令可以被拆分为：

  1. IF 阶段：根据 PC 读取指令，并计算 PC + 4；

  2. ID 阶段：译码指令，读取 rs1；

  3. EX 阶段：计算访存地址 rs1 + imm，结果保存到 exu_addr；

  4. MEM 阶段：使用 exu_addr 访问数据存储器；

  5. WB 阶段：将读出的数据写回 rd。

2 各阶段完成的功能

2.1 IF 阶段：取指

IF 阶段负责从指令存储器中读取指令，并计算下一条顺序指令地址 PC + 4。

在 IF 阶段需要完成：

  • 使用 PC 访问指令存储器；

  • 计算 PC + 4；

  • 默认情况下，PC 准备更新为 PC + 4。

需要特别注意的是：

在多周期处理器中，PC 不能像单周期处理器或流水线前端那样每个周期自动加 4 并取新指令。 当前指令进入 IF 之后，需要继续经过 ID、EX、MEM、WB 等阶段，待其执行完成后，下一条指令才能重新进入 IF。

例如：

  • 普通顺序执行指令可以在 IF 阶段就更新 PC；

  • branch 指令要在 EX 阶段根据比较结果或跳转条件决定是否更新 PC；

  • jal、jalr 指令计算出跳转目标地址后需要更新 PC。

一种较简单的设计方式是：

  • 在 IF 阶段将 PC 送出取指之后，默认更新 PC 为 PC + 4；

  • 到 EX 阶段如果发现是跳转或分支指令，再改写 PC；

PC 什么时候更新？

本实验不强制规定 PC 的具体更新时机。 你可以选择在 IF 结束阶段就将 PC 更新为 PC + 4， 也可以选择在指令完成后再更新 PC。 但无论采用哪种设计，都必须保证 branch、jal、jalr 等控制流指令能够正确修改 PC。

2.2 ID 阶段：译码与读寄存器

在单周期处理器中，译码单元的输出会直接连接到执行单元。 而在多周期处理器中，由于 EX 阶段在下一个周期才会执行，因此需要把这些结果保存下来。 在译码阶段，我们输出了很多后续阶段需要的信号，例如：

  • 目的寄存器编号

  • EX 阶段所需控制信号

  • MEM 阶段所需控制信号

  • WB 阶段所需控制信号

这些信号需要在每个阶段之间通过寄存器保存下来， 保证后续 EX、MEM、WB 阶段在各自周期内都能获得属于“同一条指令”的正确数据与控制信息。

2.3 EX 阶段：执行运算或计算地址

EX 阶段负责使用 ALU 和地址加法器完成指令的核心计算。 目前我们的所有运算指令都是可以在一个周期内完成的，因此 ALU 的输入和输出都可以直接连接到 EX 阶段的寄存器。

如何跳过不必要的阶段

本实验不强制规定 PC 的具体更新时机。 你可以选择在 IF 阶段就将 PC 更新为 PC + 4， 也可以选择在指令完成后再更新 PC。 但无论采用哪种设计，都必须保证 branch、jal、jalr 等控制流指令能够正确修改 PC。

一个比较好的方式是在 EX 阶段就计算出跳转目标地址，并在 EX 阶段根据指令类型和比较结果决定是否更新 PC。 flush 信号可以在 EX 阶段生成，表示当前指令是分支或跳转指令，需要在 EX 阶段就更新 PC。 如果 flush 信号为真，则在 EX 阶段就把 PC 更新为跳转目标地址，否则在 IF 阶段默认更新 PC 为 PC + 4。

2.4 MEM 阶段：访存

MEM 阶段负责处理 load/store 访存操作。

相比普通 ALU 指令，load 指令通常需要更多周期。 但我们会在下次实验中，引入总线和更真实的存储器行为，来模拟 load/store 指令的访存延迟。

详细请看 SUAT_sram.v 的存储器实现方式， 目前我们先假设读操作是组合逻辑实现的，给出访存地址，立即读出存储器数据。 写操作是在时钟上升沿把数据写入存储器的，因此写操作需要下一个时钟周期完成。

需要注意的是：并不是所有指令都需要访存，因此你是否可以跳过 MEM 阶段呢？

2.5 WB 阶段：写回寄存器

WB 阶段负责把指令的结果写回寄存器堆。 WB 阶段结束后，当前指令的执行全部完成。 此时处理器可以认为该指令“提交”成功，并允许下一条指令开始新的 IF 阶段。

需要注意的是：并不是所有指令都需要写回寄存器，因此你是否可以跳过 WB 阶段呢？

3 设计实现

实际实现中，我们如何保证多周期处理器在一条指令的不同阶段正确地传递数据呢？ 我们可以通过一个状态机来指示当前指令处于哪个阶段，并根据状态机的状态来控制级间寄存器的更新，以此控制数据的传递。 比如 IFU 和 IDU 之间的级间寄存器需要在某个特定的状态下更新，以保证 IF 阶段的信息能够正确传递到 ID 阶段。 同样的，别的阶段之间的级间寄存器也需要在特定状态下更新，以保证信息的正确传递。

同时，状态机还可以根据 IDU 的译码结果来决定下一条指令需要经历哪些阶段，从而控制状态机的状态转移。 例如 ADDI 指令可以选择跳过 MEM 阶段，如果跳过 MEM 阶段，控制信号和数据应该如何传递到 WBU 呢？

因此，本次实验你需要使用级间寄存器，还需要能够控制级间寄存器的更新时机，以保证数据在不同阶段之间正确传递。 在代码框架中，我们添加了一些控制信号，下面这些控制信号是不能被移除的：

  • if_id_valid：表示 IF 阶段的指令已经准备好，可以传递到 ID 阶段；

  • id_ex_valid：表示 ID 阶段的指令已经准备好，可以传递到 EX 阶段；

  • ex_mem_valid：表示 EX 阶段的指令已经准备好，可以传递到 MEM 阶段；

  • mem_wb_valid：表示 MEM 阶段的指令已经准备好，可以传递到 WB 阶段；

  • flush：表示当前指令是分支或跳转指令，需要在 EX 阶段就更新 PC;

  • wbu_commit: 表示当前指令在 WB 阶段完成提交，可以允许下一条指令进入 IF 阶段；同时 wbu_commit 也是用来 Debug 的一个重要信号，在仿真中可以通过观察 wbu_commit 信号来判断指令是否提交正确的值。

实现多周期处理器

我们的代码框架中，给出了实现多周期处理器必要的关键控制信号， 请你根据自己的理解来实现多周期处理器的行为，当然修改框架代码或者增减控制信号都是可以的。 我们几乎没有修改每个阶段内部的实现，级间寄存器和控制信号都在top.v中定义。

多周期处理器性能

多周期处理器缩短了单个时钟周期，但一条指令需要多个周期才能完成。 那么多周期处理器一定比单周期处理器快吗？ 请从时钟周期长度和 CPI 两个角度进行分析。
