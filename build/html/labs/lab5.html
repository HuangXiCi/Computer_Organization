处理器设计三

本次实验中，我们将把前面已经实现的单周期处理器改造成多周期处理器。

在单周期处理器中，一条指令必须在一个时钟周期内完成取指、译码、执行、访存和写回等所有操作。 这会导致一个时钟周期必须足够长，能够覆盖最慢指令的数据通路延迟。 例如 lw 指令需要经历取指、译码、地址计算、访存、写回寄存器等步骤， 它通常会决定整个单周期处理器的最长关键路径。

多周期处理器的基本思想是：将一条指令拆分成多个阶段执行，每个阶段占用一个时钟周期。 这样可以缩短单个时钟周期的组合逻辑延迟，并且能够在不同阶段复用硬件资源，例如 ALU 可以既用于 PC + 4， 也可以用于访存地址计算，还可以用于普通算术逻辑运算。

需要注意的是，本次实验实现的是多周期处理器，而不是流水线处理器。 多周期处理器在同一时刻仍然只处理一条指令，只是这条指令需要多个周期完成。 流水线处理器则允许多条指令同时处于不同执行阶段，这不是本次实验的内容。

1 从单周期到多周期

单周期处理器的特点是每条指令都在一个周期内完成。 因此，不同类型指令虽然实际需要的功能不同，但它们都必须占用同样长的周期。

例如：

  • add 指令只需要取指、译码、执行和写回；

  • sw 指令需要取指、译码、计算地址和写存储器；

  • lw 指令需要取指、译码、计算地址、读存储器和写回。

在单周期处理器中，所有这些操作都被放在一个周期中完成，导致时钟周期由最慢的指令决定。

多周期处理器会把指令执行过程拆成若干阶段：

  1. IF：Instruction Fetch，取指；

  2. ID：Instruction Decode，译码并读取寄存器；

  3. EX：Execute，执行运算或计算地址；

  4. MEM：Memory Access，访存；

  5. WB：Write Back，写回。

不同指令所需的阶段数量不同。 例如普通 ALU 指令不需要 MEM 阶段，而 sw 指令不需要 WB 阶段。

多周期处理器不是流水线处理器

多周期处理器虽然也把指令拆成 IF、ID、EX、MEM、WB 等阶段， 但它在同一时刻只执行一条指令。当前指令没有执行完成之前，下一条指令不会进入处理器。 因此本次实验不需要考虑流水线冒险、数据前递、流水线暂停等问题。

1.1 多周期处理器中的状态寄存器

在单周期处理器中，一条指令的所有组合逻辑都在一个周期内完成，中间结果不需要保存。 而在多周期处理器中，一条指令会跨越多个时钟周期执行，因此我们必须在阶段之间保存中间结果。

这些寄存器的作用是把单周期处理器中的长组合逻辑路径切分成多段。 每个阶段只完成其中一部分工作，并把结果保存到寄存器中，供后续阶段使用。

例如 lw rd, imm(rs1) 指令可以被拆分为：

  1. IF 阶段：根据 PC 读取指令，并计算 PC + 4；

  2. ID 阶段：译码指令，读取 rs1；

  3. EX 阶段：计算访存地址 rs1 + imm，结果保存到 ALUOut；

  4. MEM 阶段：使用 ALUOut 访问数据存储器；

  5. WB 阶段：将读出的数据写回 rd。

2 各阶段完成的功能

2.1 IF 阶段：取指

IF 阶段负责从指令存储器中读取指令，并计算下一条顺序指令地址 PC + 4。

在 IF 阶段需要完成：

  • 使用 PC 访问指令存储器；

  • 计算 PC + 4；

  • 默认情况下，PC 准备更新为 PC + 4。

需要注意的是，在多周期处理器中，PC 不能像单周期处理器那样每个周期都自动加 4。 PC 只有在合适的阶段才能更新。 例如：

  • 普通顺序执行指令可以在 IF 阶段就更新 PC；

  • branch 指令要在 EX 阶段根据比较结果或跳转条件决定是否更新 PC；

  • jal、jalr 指令计算出跳转目标地址后需要更新 PC。

一种较简单的设计方式是：

  • 在 IF 阶段将 PC 送出取指之后，默认更新 PC 为 PC + 4；

  • 到 EX 阶段如果发现是跳转或分支指令，再改写 PC；

PC 什么时候更新？

本实验不强制规定 PC 的具体更新时机。 你可以选择在 IF 阶段就将 PC 更新为 PC + 4， 也可以选择在指令完成后再更新 PC。 但无论采用哪种设计，都必须保证 branch、jal、jalr 等控制流指令能够正确修改 PC。

2.2 ID 阶段：译码与读寄存器

在单周期处理器中，译码单元的输出会直接连接到执行单元。 而在多周期处理器中，由于 EX 阶段在下一个周期才会执行，因此需要把这些结果保存下来。

2.3 EX 阶段：执行运算或计算地址

EX 阶段负责使用 ALU 和地址加法器完成指令的核心计算。 目前我们的所有运算指令都是可以在一个周期内完成的，因此 ALU 的输入和输出都可以直接连接到 EX 阶段的寄存器。

我们在执行 Load/Store 指令时，我们可以尝试跳过 MEM 阶段，直接进行写回阶段。

如何跳过 MEM 阶段

本实验不强制规定 PC 的具体更新时机。 你可以选择在 IF 阶段就将 PC 更新为 PC + 4， 也可以选择在指令完成后再更新 PC。 但无论采用哪种设计，都必须保证 branch、jal、jalr 等控制流指令能够正确修改 PC。
